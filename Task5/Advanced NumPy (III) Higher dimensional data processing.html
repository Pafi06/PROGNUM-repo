5.3. To higher dimensions

So far, we have been mostly working on one dimensional data structures, i.e. one dimensional array. As an astronomer, we often work on one-dimensional data like light curve and spectroscopic data. Furthermore, we also deal with images which are 2-dimensional data structure, even data cubes, e.g. Integral Field Units (IFU) which is in 3-dimensional data structure. If a telescope can also distinguish polarization, we even need a 4-dimensional data structure. In this section we learn how to create arrays with a shape and show how to visualize 2-dimensional data with Matplotlib methods.

5.3.2. Concatenation of arrays

Often you have a situation where you need more space in an array because some elements need to be appended. In the notebook we discussed function numpy.append() which allows to append one or more values to an array and creates a new array for the result. If you have multiple arrays to append, then the function numpy.concatenate() is more convenient:

import numpy as np

x1 = np.array([1,2,3])
x2 = np.array([4,5,6])
x3 = np.array([7,8,9])

X = np.concatenate([x1,x2,x2])
print(X)

5.4. NumPy array attributes and methods




5.4.1. An overview of array attributes


Some common ones are listed below. A complete overview of attributes and methods of NumPy arrays can be found at numpy.ndarray.








Attribute




Description










ndim




The number of axes (dimensions) of the array. In the Python world, the number of dimensions is referred to as rank.








shape




The dimensions of the array. This is a tuple of integers indicating the size of the array in each dimension. For a matrix with n rows and m columns, shape will be (n,m). The length of the shape tuple is therefore the rank, or number of dimensions, ndim.








size




The total number of elements of the array. This is equal to the product of the elements of shape.








dtype




An object describing the type of the elements in the array. One can create or specify dtype’s using standard Python types. Additionally NumPy provides types of its own. numpy.int32, numpy.int16, and numpy.float64 are some examples.








itemsize




The size in bytes of each element of the array. For example, an array of elements of type float64 has itemsize 8 (=64/8), while one of type complex32 has itemsize 4 (=32/8). It is equivalent to ndarray.dtype.itemsize.








nbytes




Total number of bytes consumed by the elements of the array.








data




The buffer containing the actual elements of the array. Normally, we won’t need to use this attribute because we will access the elements in an array using indexing facilities.








flags




Information about the memory layout of the array.

5.5. 3D data structures


As an astronomer, the most common data we use are images, which is two dimensional. However, radio data and IFU data (Integral Field Units) have three dimensional structures (datacube). These data are also taken at different frequencies which form the third dimension on an 2D images (see illustration below).


In this section we show how to use NumPy arrays to analyze and process astronomical data. The axes x, y and z correspond usually with R.A. (Right Ascension), Dec. (Declination) and Freq. (frequency).
